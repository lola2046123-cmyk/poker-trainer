# 移动端无法正常显示 — 排查说明

部署到 GitHub 后 **Web 端正常、手机端异常** 时，按下面顺序排查。

---

## 1. 视口与全屏高度（iOS Safari 常见）

**现象**：白屏、只显示一部分、底部有空白、或全屏遮罩盖不全。

**原因**：iOS Safari 对 `100dvh`（动态视口高度）存在已知 bug，计算偏大或与地址栏不同步，导致布局错乱。

**已做修改**：
- 全屏容器改用 `100vh` + `-webkit-fill-available` 兜底，再写 `100dvh`。
- `#result-page`、`#landing-page`、`body` 在样式中已加上述兜底。
- 若仍异常，可暂时在 CSS 中全局把 `100dvh` 改为 `100vh` 再试。

---

## 2. 题库脚本 404（手机能复现、电脑正常）

**现象**：首页能打开，但点「萌新/票友/懂哥」没反应，或控制台报 `allQuestions is not defined` / 网络错误。

**原因**：`questions-data.js` 使用相对路径 `questions-data.js`。若访问地址带目录而文件不在同一目录，手机浏览器会请求错路径导致 404。

**检查**：
- GitHub 仓库根目录是否有 `questions-data.js`（与 `index.html` 同级）。
- 访问方式：
  - **Project Pages**：`https://<user>.github.io/<repo>/` → 脚本应来自 `https://<user>.github.io/<repo>/questions-data.js`。
  - 若用 **docs 目录** 发布：`index.html` 和 `questions-data.js` 都要在 `docs/` 下，且访问为 `https://<user>.github.io/<repo>/`（指向 docs）。

**已做修改**：若 `allQuestions` 或 `CHAPTER_CONFIG` 未定义，页面会显示「题库加载失败」提示，便于确认是脚本未加载。

---

## 3. CDN 资源在移动网络失败

**现象**：电脑 WiFi 正常，手机 4G/5G 下白屏或样式错乱。

**原因**：Tailwind / confetti / html2canvas / animate.css 等从 CDN 加载，部分运营商或地区会慢、被墙或拦截。

**排查**：
- 手机浏览器打开：  
  `https://cdn.tailwindcss.com`  
  `https://cdn.jsdelivr.net/npm/canvas-confetti@1.9.3/dist/confetti.browser.min.js`  
  看是否能正常打开。
- 若 CDN 不可用：可把上述资源下载到仓库同目录，改为相对路径引用（需同时改 `index.html` 中的地址）。

**已做修改**：Tailwind 未加载时保留 `no-tailwind` 兜底样式；并增加延迟检测，尽量在加载完成后移除 `no-tailwind`，减少「一直用兜底样式」的情况。

---

## 4. 点击/触摸无反应（300ms 延迟或点击穿透）

**现象**：点按钮要按很久才有反应，或误触。

**已做修改**：在 `<body>` 上增加了 `touch-manipulation`（Tailwind），即 `touch-action: manipulation`，减少 300ms 延迟并降低误触。

---

## 5. 安全区域与刘海屏

**已做修改**：
- viewport 增加 `viewport-fit=cover`，并增加 Apple 全屏相关 meta。
- 结算页底部按钮区使用 `padding-bottom: max(1.5rem, env(safe-area-inset-bottom))`，避免被底部横条遮挡。

---

## 6. 建议自检清单（手机实测）

| 项目 | 操作 |
|------|------|
| 访问地址 | 用手机浏览器直接打开 `https://<user>.github.io/<repo>/`（或你实际发布地址） |
| 控制台 | 手机用 Safari/Chrome 远程调试或 vConsole 等，看是否有 404、CORS、JS 报错 |
| 无痕/隐私模式 | 关掉无痕再试（部分浏览器隐私模式会限制本地存储或脚本） |
| 缓存 | 清除站点数据或强制刷新后再打开 |
| 网络 | 换 WiFi / 4G 对比，排除某网络下 CDN 或 DNS 问题 |

---

## 7. 已修改文件摘要

- **index.html**
  - viewport 增加 `viewport-fit=cover`，Apple 全屏 meta。
  - 全屏相关高度：`100dvh` 保留，并增加 `100vh`、`-webkit-fill-available` 兜底。
  - `#result-page` 的 HTML 类改为 `min-h-screen`，避免在旧版 iOS 上只用 `100dvh`。
  - 结算页底部增加 `env(safe-area-inset-bottom)`。
  - body 增加 `touch-manipulation`。
  - Tailwind 加载：增加延迟检测与再次尝试移除 `no-tailwind`。
  - 检测 `allQuestions` / `CHAPTER_CONFIG`，缺失时显示「题库加载失败」并提示检查 `questions-data.js` 路径。

若按上述排查后仍有问题，可把**具体现象**（白屏 / 只显示一部分 / 点不动 / 报错文案）和**手机型号与系统、浏览器**发出来，便于进一步定位。
